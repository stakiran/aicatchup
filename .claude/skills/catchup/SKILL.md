# catchup — AI情報 週次キャッチアップ

`.claude/rules/source.md` に列挙されたソースを巡回し、**今週(ISO週)の新着ネタ**を集めて
日本語の週次ダイジェストにまとめるスキル。全面日本語で出力する（英語は使わない）。

## 方針（ユーザーの要件）

- **週単位**で追う。対象は「今日が属するISO週」。
- 各ソース **最大5件**。反響(upvote等)が多い／インパクトが強そうなものを優先選定。
- すでに `seen.json` にあるネタは**再掲しない**（差分のみ）。
- 新着が無いソースは「更新なし」と一行だけ書いて**すぐ終わる**。
- 出力は**全面日本語**（ヘッドラインも概要も日本語に訳す）。

## 手順

1. **今週を決める**
   - 今日の日付からISO週コードを算出（例: `2026-06-20` → `2026-W25`）。
   - 週の月曜〜日曜の日付範囲も出す（ダイジェストのタイトル用）。

2. **ソースを読む**
   - `.claude/rules/source.md` のURL一覧を取得。
   - 各URLを WebFetch で取得し、ネタ一覧（タイトル・リンク・日付・反響数）を抽出。
     - WebFetch が 403 等で失敗したら WebSearch（`allowed_domains` でそのドメイン指定）にフォールバック。
   - HuggingFace の `…/papers/week/YYYY-Www` は、URL中の週コードを**今週**に置き換えてから取得。

3. **絞り込み**
   - 各ネタのIDを作る（`anthropic:<slug>` / `openai:<slug>` / `hf:<paper-id>` の形式）。
   - `seen.json` の `seen[].id` に**既出のものを除外**。
   - 対象期間は今週。Anthropic/OpenAIは記事日付が今週のもの。HuggingFaceは週ページ自体が今週分。
   - 残ったネタを**インパクト順**に並べ、各ソース**上位5件**を採用。
     - HuggingFace: upvote数で機械的にランキング。
     - Anthropic/OpenAI: 反響数が取れないので内容のインパクトで判断（新モデル・大型提携・安全性/規制・主要プロダクト更新を上位に）。

4. **書き出し**
   - `docs/weeks/<週コード>.md` に追記（無ければ新規作成）。フォーマットは既存の週ファイルに合わせる
     （ソース別の見出し → 各ネタ「番号. ヘッドライン / リンク(日付) / 概要2〜3行」）。
   - 採用した全ネタのIDを `seen.json` の `seen` 配列に追記（`id` / `week` / `title`）。
   - `docs/index.md` の「週次ダイジェスト（新しい順）」に、その週がまだ無ければ**先頭**にリンク追記。

5. **更新が無い場合**
   - あるソースで新着ゼロなら、そのソース見出しの下に「※ 今週の新着なし」とだけ書く。
   - 全ソース新着ゼロなら、ファイルは変更せず「今週は新着なし」とユーザーに報告して終了。

## ファイル構成

- `.claude/rules/source.md` … 追うソースのURL一覧（ユーザーが編集）
- `seen.json` … 取得済み台帳（重複防止）
- `docs/index.md` … 目次（GitHub Pages トップ）
- `docs/weeks/<週コード>.md` … 週次ダイジェスト本体

## 注意

- 個別記事URLが特定できないネタは、`…/news/` 等の一覧ページURLを暫定で使い、その旨を脚注で添える（推測URLを断定で書かない）。
- 概要はネタの中身が分かる範囲で簡潔に。原文の引き写しではなく日本語要約にする。
